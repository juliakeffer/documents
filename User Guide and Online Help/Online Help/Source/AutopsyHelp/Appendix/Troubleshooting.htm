Troubleshooting

This section provides information to help you understand error messages you may see in Autopsy, possible causes, and suggested solutions.

Error Message

Possible Cause

Explanation
When adding an image to a case, the message Error extracting file from image: (ntfs_uncompress_compunit: Phrase token offset is too lart: 1 (max:0) appears. This message occurs when trying to process a compressed NTFS file that is corrupt. This can occur when a file has been deleted and the clusters have been reallocated to another file. Compare the MD5 sum of the file you received with that of the original file. If it matches, the file is corrupt and you need to capture a new image from the original disk. If the MD5 sum does not match, the image was corrupted in transit. Ask the investigator who created the image to send it to you again.
Options window shows "Error: Index does not exist" After you add a hash database, Autopsy needs to create an index to use it. In the Options window, click Index.
The ingest message list shows "No known bad database set" You did not supply a hash database that lists known bad files. If you want Autopsy to analyze known bad files, you need to provide separate databases for both known good and bad files. If a database contains both sets of definitions, such as the NIST NSRL database, you must designate it as good or bad. You may find information on the Internet about methods to separate the database into two parts.
In the Result Viewer, the file status of all the files is "unknown". You did not add a hash database to categorize files as known or unknown. If you did not add a hash database, follow the steps in Adding a Hash Database to add one.
You did not identify the hash database you added as NSRL, which indicates that the files in the database are known good files. You must identify a hash database as either NSRL or Known Bad. A single hash database cannot be designated as both types. Either you need to add a hash database for known good files (NSRL), or change the type of database you already added.
