About Disk Analysis

Autopsy refers to the process of automatically analyzing the disk contents as ingest The process of extracting and classifying information from the image file added to Autopsy.. Ingest The process of extracting and classifying information from the image file added to Autopsy. extracts the most common types of information needed from a disk image, which avoids the need to perform the tasks manually. Autopsy processes the user-related files first, to find the most likely sources of interesting information. The process runs in the background, which means you can begin to browse the data while the process continues.

In this section you will learn about the types of ingest The process of extracting and classifying information from the image file added to Autopsy. modules, how to configure ingest The process of extracting and classifying information from the image file added to Autopsy., and how to stop and restart ingest The process of extracting and classifying information from the image file added to Autopsy.. At the end of this section, you will know how to customize ingest The process of extracting and classifying information from the image file added to Autopsy. to analyze the type of information you want to know about in the disk image.

The ingest The process of extracting and classifying information from the image file added to Autopsy. process uses modules that analyze specific types of data. There are six default modules that you can activate depending on the type of data you want analyzed.

Ingest Modules

Module Name Purpose
Keyword Search

Creates an index of keywords using Apache SOLR search server and enables you to perform simple keyword searches and define keyword lists that are used during ingest to produce search results.

Hash Database

Calculates the MD5 Message Data 5 – a one way hash function hash of each file and can lookup it up in the and one or more hash databases. Categorizes files as known or unknown types, based on the contents of the hash database you supplied.

Thunderbird Parser Extracts the contents of email folders from Outlook and Thunderbird email clients.
Exif Parser Analyzes the metadata Properties of a file attached to the file by some programs (location, date, and device used to capture the image) for image files, in formats such as JPG Graphics file type/extension (lossy compressed 24 bit color image storage format developed by the Joint Photographic Experts Group) and PNG Left panel of main window.
Recent Activity

Extracts recent user activity by focusing on web artifacts and system settings.

·       Extracts information about the last seven days of disk activity

·       Categorizes files based on the file type metadata Properties of a file attached to the file by some programs

·       Extracts Internet artifacts, such as downloads, browsing history, installed programs, and search engine queries

·       Determines the device IDs of any devices connect to the computer at the time the disk image was captured

The module only supports Windows file systems.

You can also select whether to analyze the unallocated disk space. This is the area that the operating system has marked as available for use, but which it has not yet overwritten with new data. If you choose not process it, ingest The process of extracting and classifying information from the image file added to Autopsy. will finish more quickly, but you may miss some information.

The size of the image and the ingest The process of extracting and classifying information from the image file added to Autopsy. modules you use determine how long it will take to analyze the image contents. If you are not sure what data you need, you should select all the available modules. If the process takes too long, you can stop it and deselect some modules. Alternatively, if you think you only need certain types of information, you can limit the modules you select, and if you need more information, you can rerun ingest The process of extracting and classifying information from the image file added to Autopsy. later. For example, if you think you only need Internet artifacts, you may not need to select the Hash Lookup module.

How do I speed up Ingest?

You can disable some ingest modules, such as Hash DatabaseY, which can take a long time to run on a large disk image. You can also exclude processing the unallocated disk space, although you may miss some important information. See Configuring Ingest for more information.

I started ingest through the wizard, but I want to add or remove modules. How do I stop it and restart it?

You can stop Ingest by right-clicking in the bottom right corner, where you can see the blue progress bar. After you reconfigure Ingest, you can restart it by right-clicking the image name in the left panel. See Stopping and Restart Ingest for more information.
